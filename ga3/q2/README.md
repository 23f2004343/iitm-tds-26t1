# Q2: LLM Structured Output - FastAPI Sentiment Analysis

## Task

Create a FastAPI web server endpoint `POST /comment` that accepts a JSON payload containing a text comment and returns a structured JSON object detailing the comment's sentiment and a 1-5 rating. The backend must leverage the `gpt-4o-mini` model to fulfill these structural requirements directly from the LLM.

---

## Requirements

* Create a FastAPI web server running locally on port 8000.
* Provide an endpoint `POST /comment` conforming to `application/json`.
* Request format must supply a single `comment` string field.
* Response format must strictly return two fields: `sentiment` (restricted to `"positive"`, `"negative"`, or `"neutral"`) and `rating` (an integer extending from `1` to `5`).
* Connect `gpt-4o-mini` via an OpenAI-compatible API client to perform the inference.
* Implement error handling and rate-limit mitigation (HTTP 429) to ensure graceful 500 mitigation and strict schema enforcement.
* Use `ngrok` to expose the API to the public internet for automated grader testing.

---

## Approach

### 1. Build the REST Interface
Utilize the FastAPI Python library, along with `uvicorn` and `pydantic`, to construct a lightweight ASGI web server. Pydantic's `BaseModel` handles the serialization, validation, and deserialization of the incoming request JSON and outgoing response JSON.

### 2. Connect to the LLM (OpenAI)
Integrate the official `openai` Python SDK. The `beta.chat.completions.parse` method is natively utilized to enforce strict JSON schemas (`response_format`) mapped directly to the defined `SentimentResponse` Pydantic model. 

### 3. Mitigate LLM Artifacts
Large language models (even with structured schema instructions) can occasionally inject unexpected capitalization (e.g., `Positive`) or synonymous terms (e.g., `mixed`). 
To guarantee passing test cases, wrap the final parsed response in a strict pipeline that downcases the output and intercepts any string outside the strict `["positive", "negative", "neutral"]` array, defaulting it to `"neutral"`.

### 4. Create Public Tunnel
Expose the locally hosted instance to the public internet using `ngrok` to facilitate remote webhook integration for the auto-grader platform.

## Installation

**Install Dependencies:**
```powershell
pip install fastapi uvicorn pydantic openai
```

**Ngrok Setup:**
Ensure `ngrok` is installed on your local machine and your authentication token is configured:
```powershell
ngrok config add-authtoken YOUR_AUTHTOKEN_HERE
```

## Execution

### Step 1: Start FastAPI Server (Terminal 1)

**Define Environment Variable:**
Set your OpenAI API key (or OpenAI-compatible key, like OpenRouter) so the pipeline can authenticate.
```powershell
$env:OPENAI_API_KEY="sk-..."
```

**Run Uvicorn Server:**
```powershell
python main.py
```
*(Alternatively: `uvicorn main:app --host 0.0.0.0 --port 8000`)*

**Expected Output:**
```
INFO:     Started server process [22592]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### Step 2: Start ngrok Tunnel (Terminal 2)

**Create Public Tunnel:**
```powershell
ngrok http 8000
```

**Expected Output:**
